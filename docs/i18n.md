# Internacionalització (i18n)

Claqueta utilitza [next-intl](https://next-intl.dev) per gestionar les traduccions.
L'idioma per defecte és el català (`ca`).

## Com afegir un nou idioma

1. **Copia el fitxer de missatges:**
   ```bash
   cp messages/ca.json messages/xx.json
   ```
   on `xx` és el codi ISO 639-1 de l'idioma (per exemple, `es`, `en`, `fr`).

2. **Tradueix les claus:** Obre `messages/xx.json` i tradueix tots els valors.
   Les claus (noms dels camps) no es toquen — només els valors.
   Tingues en compte els plurals ICU MessageFormat:
   ```json
   "pending": "{count, plural, one {Tienes # episodio pendiente} other {Tienes # episodios pendientes}}"
   ```

3. **Registra l'idioma:** Obre `src/i18n/config.ts` i:
   - Afegeix el codi a l'array `locales`.
   - Afegeix el nom visible a `localeNames`.

   ```ts
   export const locales = ["ca", "es"] as const;

   export const localeNames: Record<Locale, string> = {
     ca: "Català",
     es: "Español",
   };
   ```

4. **Importa els missatges:** Obre `src/i18n/request.ts` i afegeix la importació:
   ```ts
   import es from "../../messages/es.json";

   const messages: Record<string, typeof ca> = { ca, es };
   ```

5. **Ja està!** L'idioma apareixerà automàticament al selector de
   Configuració → Idioma.

## Convencions

- **Claus en anglès** amb format `camelCase` (p. ex. `searchCta`, `markWatched`).
- **Namespaces** per àrea: `auth`, `nav`, `pages.series`, `pages.calendar`, etc.
- **Plurals** amb ICU MessageFormat: `{count, plural, one {# sèrie} other {# sèries}}`.
- **Dates i temps relatius** es formaten amb `Intl.DateTimeFormat` /
  `Intl.RelativeTimeFormat` passant el locale de l'usuari — no s'inclouen
  als fitxers de traducció.
- **No hardcodejar text visible** als components. Tot ha de sortir de
  `useTranslations` (client) o `getTranslations` (server).
